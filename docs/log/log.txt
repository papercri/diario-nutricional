 5. UTILIDADES MATEMÁTICAS (MIFFLIN-ST JEOR)
================================================================================

Archivo: src/utils/mifflinStJeor.ts

Fórmulas implementadas:

  BMR (hombres):  10 × peso(kg) + 6.25 × altura(cm) - 5 × edad(años) + 5
  BMR (mujeres):  10 × peso(kg) + 6.25 × altura(cm) - 5 × edad(años) - 161

  TDEE = BMR × multiplicador de actividad:
    sedentary:   1.2
    light:       1.375
    moderate:    1.55
    active:      1.725
    very_active: 1.9

  Meta calórica = TDEE × ajuste de objetivo:
    lose:     0.80 (déficit)
    maintain: 1.00 (mantenimiento)
    gain:     1.15 (superávit)

  Macros:
    Proteínas: 30% de calorías / 4 kcal/g
    Grasas:    25% de calorías / 9 kcal/g
    Carbohidratos: 45% de calorías / 4 kcal/g

Exportaciones:
  - calculateBMR(weight, height, age, sex): number
  - calculateTDEE(bmr, activityLevel): number
  - calculateCalorieGoals(profile: UserProfile): CalorieGoals

 6. SERVICIOS DE API
================================================================================

6.1. src/services/openFoodFacts.ts
  - searchFood(query, page=1): busca alimentos en Open Food Facts
    URL: https://world.openfoodfacts.org/cgi/search.pl?search_terms={query}&json=true
  - getProductByBarcode(barcode): obtiene producto por código de barras
    URL: https://world.openfoodfacts.org/api/v2/product/{barcode}.json
  - Tipado estricto de todas las respuestas
  - Manejo de errores con try/catch (se captura en los stores/vistas)

6.2. src/services/tipsService.ts
  - fetchTip(): obtiene un consejo de bienestar
  - Fuente primaria: Advice Slip API (https://api.adviceslip.com/advice)
  - Fuente secundaria: array de 15 consejos curados en español (fallback)
  - Cache de 1 hora: evita llamadas repetidas a la API
  - Retorna { tip: string, source: 'api' | 'cache' | 'fallback' }

 16. FEATURE — AI NUTRITION ANALYZER (2026-07-17)
================================================================================

Se implementó un analizador de comidas potenciado por IA que permite al usuario
describir una comida en lenguaje natural y recibir un análisis nutricional
estimado completo.

16.1. Arquitectura

  Flujo: Vue App → POST /api/analyze-meal → Vercel Serverless Function → AI API
  Proveedor primario: Groq (llama-3.3-70b-versatile)
  Proveedor fallback: Cerebras (llama-3.3-70b)

  Si Groq falla, se intenta automáticamente con Cerebras.
  API keys protegidas en variables de entorno del servidor.

16.2. Backend — Vercel Serverless Functions

  api/analyze-meal.ts
    - Handler principal: valida request POST, llama providers, valida respuesta AI
    - System prompt estructurado en español con JSON schema requerido
    - Validación estricta del schema de respuesta del AI
    - Fallback automático: Groq → Cerebras

  api/providers/groq.ts
    - Cliente para Groq API (https://api.groq.com/openai/v1/chat/completions)
    - Modelo: llama-3.3-70b-versatile
    - API key desde process.env.GROQ_API_KEY
    - Temperature: 0.3, max_tokens: 1024

  api/providers/cerebras.ts
    - Cliente para Cerebras API (https://api.cerebras.ai/v1/chat/completions)
